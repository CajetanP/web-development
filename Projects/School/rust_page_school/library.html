Standardowa Biblioteka

Organizacja

Standardowa biblioteka Rust jest zorganizowana w tzw. crates. Pojedynczy crate jest odpowiednikiem biblioteki w większości języków oraz jest podzielony na modules, z których każdy zajmuje się jednym konkretnym zadaniem. Biblioteka, jak cały język, nieustannie się rozwija i niektóre jej części są niestabilne. Mimo to do użytku dostajemy ogrom użytecznych funkcji, struktur i typów, których znajomość pomaga w tworzeniu programów.

core

Crate core to najważniejsza część biblioteki Rusta. Jest podstawą standardowej biblioteki i nie jest zależny ani od systemu operacyjnego, ani innych bibliotek. Funkcjonalność jaką oferuje zawiera między innymi makra, takie jak assert, panic czy write oraz standardowe typy danych, np. i32, f64, u8, a także funkcjonalność taką jak borrowing, clone i fmt.

Istotne części core crate:

  • any - module implementujący trait Any
  • borrow - module do pracy z "pożyczonymi" danymi
  • cell - współdzielony, mutable "pojemnik" na dane
  • char - manipulacja znakami
  • clone - trait Clone dla typów które nie mogą być automatycznie kopiowane
  • cmp - funkcjonalność do implementacji porównywania typów
  • mem - podstawowe funkcje do zarządzania pamięcią
  • ptr - surowe wskaźniki, *const T i *mut T
  • slice - korzystanie z fragmentów stringów lub tablic
  • sync - podstawowa funkcjonalność synchronizacji

std

Crate std jest największą częścią biblioteki, w nim jest zawarta większość funkcjonalności i właśnie jego najczęściej się używa. Zawiera typy takie jak Vec, Option, Box, Result, operacje na prymitywnych typach, standardowe makra, I/O, operacje wielowątkowe i wiele więcej. std jest automatycznie dostępne w każdym programie.

Istotne części std crate:

  • boxed - typ wskaźnika do wygodnego alokowania zmiennych na heap
  • env - interakcja ze środowiskiem procesu
  • error - Traits do zarządzania błędami programu
  • fs - operacje do manipulacji systemem plików
  • net - podstawowa funkcjonalność do komunikacji z użyciem TCP/UDP
  • option - opcjonalne zmienne
  • os - funkcjonalność właściwa danemu systemowi operacyjnemu
  • process - module do zarządzania procesami
  • string - kodowany w UTF-8, rozszerzający się string
  • thread - natywne wątki
  • vec - rozszerzająca się tablica, alokowana na heap

collections

Crate collections zawiera struktury danych - kolekcje które są użyteczne podczas tworzenia praktycznie wszystkich rodzajów programów. Na ten moment część funkcjonalności jest dzielona z std::collections. Możemy w nim znaleźć chociażby binary heap oraz linked list.

Istotne części collections crate:

  • binary_heap - priority queue implementowane z użyciem binary heap
  • btree_map - map bazowana na B-Tree
  • btree_set - set bazowany na B-Tree
  • linked_list - doubly-linked list z owned nodes
  • vec_deque - double-ended queue implementowana z użyciem vector

alloc

Crate alloc to biblioteka najniższego poziomu oferująca funkcjonalność związaną z zarządzaniem pamięcią. Przeznaczony jest głównie do budowania innych bibliotek, nie do pisania programów użytkowych. Jej zawartość jest reeksportowana w crate std.

Istotne części alloc crate:

  • rc - jednowątkowy, liczący referencje pointer oferujący shared ownership
  • arc - atomic reference counter - rc przeznaczony do obsługi wielu wątków
    • heap - eksperymentalny module z funkcjami alokującymi i dealokującymi pamięć
